# 🤖 LLMs & AI/ML Engineering

From-scratch implementations, courses, and tooling for building with and on top of LLMs.

## Build From Scratch

- **[LLMs from Scratch (Sebastian Raschka)](https://github.com/rasbt/LLMs-from-scratch)** — Code-along book that builds a GPT-style model end-to-end. Best single resource for understanding what's actually inside an LLM.
- **[Reasoning from Scratch (Raschka)](https://github.com/rasbt/reasoning-from-scratch)** — Companion repo focused on chain-of-thought and reasoning techniques.
- **[Understanding LLMs Using Middle School Math](https://towardsdatascience.com/understanding-llms-from-scratch-using-middle-school-math-e602d27ec876/)** — Strips away jargon; useful for explaining transformers to non-technical stakeholders.

For low-level/C-based tensor implementations, see [gpu-lowlevel.md](gpu-lowlevel.md).

## Courses

- **[Chip Huyen — AI Engineering Resources](https://github.com/chiphuyen/aie-book/blob/main/resources.md)** — The companion resource list to her AI Engineering book.
- **[DeepLearning.ai — Claude Code](https://www.deeplearning.ai/short-courses/claude-code-a-highly-agentic-coding-assistant/)** — Short course on Anthropic's coding agent.
- **[DeepLearning.ai — Agentic AI](https://www.deeplearning.ai/courses/agentic-ai/)** — Full course on building agents.
- **[DeepLearning.ai — Voice Agents with Google ADK](https://www.deeplearning.ai/short-courses/building-live-voice-agents-with-googles-adk/)** — Real-time voice agent construction.
- **[Stanford CS231n](https://cs231n.stanford.edu/2017/syllabus.html)** — Convolutional networks for visual recognition. The course that trained a generation of CV engineers.
- **[Stanford CS229 (YouTube)](https://m.youtube.com/playlist?list=PLoROMvodv4rOY23Y0BoGoBGgQ1zmU_MT_)** — Andrew Ng's full ML course.
- **[Understanding Deep Learning](https://udlbook.github.io/udlbook/)** — Free textbook by Simon Prince; modern, well-illustrated.
- **[Microsoft GenAI on Databricks](https://learn.microsoft.com/en-us/training/paths/implement-generative-ai-engineering-azure-databricks/)** — Azure-specific GenAI engineering path.

## Fine-Tuning & Optimization

- **[Unsloth](https://unsloth.ai/)** — 2x faster fine-tuning, ~70% less memory. Reach for this before paying for more GPU.
- **[Unsloth GitHub](https://github.com/unslothai/unsloth)** — Source and recipes for Mistral, Gemma, Llama.
- **[Deploy LLMs to Phone (Unsloth)](https://unsloth.ai/docs/new/deploy-llms-phone)** — On-device deployment guide.
- **[Flash Attention Explained](https://medium.com/@lsj3285007/flash-attention-6cab389e5bf0)** — Why attention got fast.

## Visual Explainers

- **[MLU Explain](https://mlu-explain.github.io/)** — Visual, interactive ML concept explainers from AWS.
- **[RNN in a Spreadsheet (byhand.ai)](https://www.byhand.ai/p/recurrent-neural-network-spreadsheet)** — Build an RNN in Excel. The clearest possible explanation.

## Research & Tooling

- **[Simon Willison's `llm` CLI](https://github.com/simonw/llm)** — Talk to any model from the terminal; plugin ecosystem.
- **[LlamaGym](https://github.com/KhoomeiK/LlamaGym)** — RL fine-tuning for LLMs.
- **[PyVene (Stanford NLP)](https://github.com/stanfordnlp/pyvene)** — Mechanistic interpretability toolkit.
- **[IndicLLM Suite (AI4Bharat)](https://github.com/AI4Bharat/IndicLLMSuite)** — Models and data for Indian languages.
