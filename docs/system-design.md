# 🏗️ System Design

Designing distributed systems, data-intensive applications, and scalable architecture.

## Foundational

- **[Designing Data-Intensive Applications (DDIA)](https://github.com/marangelologic/books/blob/master/Designing%20Data%20Intensive%20Applications.pdf)** — Martin Kleppmann's book. The reference text for anyone touching data systems.
- **[DDIA — Notion Notes](https://nebulous-arrow-2b2.notion.site/Designing-Data-Intensive-Applications-Data-Warehousing-and-engineering-related-notes-169057ebad8380919fcad4361df42289)** — Dense chapter-by-chapter notes; useful for review or skipping the slow parts.
- **[DDIA Summary (pagefy.io)](https://pagefy.io/system-design/designing-data-intensive-applications-by-martin-kleppmann)** — Web-based reading companion.
- **[Grokking System Design](https://www.educative.io/courses/grokking-the-system-design-interview)** — Most-cited interview prep course; case-study format.
- **[System Design Lab](https://systemdesignlab.netlify.app/)** — Free interactive practice problems.

## Distributed Systems Deep Dives

- **[Implementing Raft (Eli Bendersky)](https://eli.thegreenplace.net/2020/implementing-raft-part-0-introduction/)** — Multi-part series implementing the Raft consensus algorithm in Go.
- **[Beej's Network Programming Guide](https://beej.us/guide/bgc/html/#audience)** — The classic. Sockets, TCP/IP, the protocols underneath everything.
- **[PostgreSQL Internals (Percona)](https://www.percona.com/blog/postgresql-internals-for-newbies-a-guide-to-data-storage-part-one/)** — How a real database stores data on disk.

## Real-World Engineering

- **[Netflix Tech Blog — Building Confidence](https://netflixtechblog.com/building-confidence-in-a-decision-8705834e6fd8)** — Decision-making under uncertainty at scale.
- **[System Design Career Story](https://medium.com/@himanshusingour7/system-design-took-me-from-missed-calls-to-28-base-package-a13bea4e3c03)** — One engineer's path from rejections to senior package; concrete prep advice.
